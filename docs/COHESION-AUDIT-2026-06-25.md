# Tsumugu Core — Cohesion Report (2026-06-25 snapshot)

Built + wired + tested vs. the working tree. Drives the patch worklist. The single most consequential fact: the borrowed `@tsumugu/engine` is mid-migration (`WordStatus` `new`/`l1..l4` → `"1".."4"`), and that half-done rename radiates failures into stores, merge, Anki, reader CSS, and the typecheck across every consuming package.

## 1. WHAT'S DONE

**Reader core**
- Token-by-token reader with ruby readings, known-state coloring, new-target marks, and keyboard grading, mounted from `main.ts`, green on 4 `reader.test.ts` cases.
- Reading/gloss extraction maps the active hv/py/zhuyin and en/vi mode onto entry data (dual-rail ruby-rebuild test).
- Click-to-open definition+form popup with lookup-is-capture, dict link, and guess-first reveal (capture test passing).
- Coupling guard confirms the reader is scrubbed of personal-monorepo references and legacy routes.
- Responsive breakpoints (rail hidden ≤980px, prose shrinks ≤720px), tested.

**App frame / backbone**

**Packs / PWA / build**
